# 分派提示词（复制给别的智能体 / 别的会话用）

把下面这段整体复制，改掉【省份】，发给一个新的 Claude Code 会话即可。一个智能体建议包 3~5 个省。

---

```
项目目录：/Users/yangrui/Downloads/公立图书馆电子资源

任务：为【广东、湖北、湖南】这几个省的【省级公共图书馆】采集免费电子资源和办证方式，
每个馆在 data/libraries/ 下建一个 JSON 文件（文件名用省份拼音，如 guangdong.json）。

开始前必须先读这三个文件，严格照其规范执行：
- 结构侦察教程.md 【重点】——怎么判断网站是SSR还是SPA、怎么找到资源列表接口、怎么翻页/去重拿到全量、怎么验收没漏。你最容易在"发现网站结构"这步卡住，这份教程就是专门解决它的，务必照着一步步做。
- 采集指南.md （数据格式、铁律、口径、省份进度表）
- data/libraries/shanghai.json 和 zhejiang.json 和 beijing.json （已完成的真实样例，照抄结构）

关键要求（详见采集指南.md，这里只强调最容易错的）：
1. 只填亲眼在官网核实到的信息，绝不编造。尤其"能否线上办证"(registration.online) 是本项目核心，务必查实。
2. access_method（馆内/馆外）有官网明确标注就填，没标就留空——绝不从URL或"镜像"字样推断。
3. 资源要全量。省级馆官网基本是 SPA，必须用真实浏览器：加载 web-access skill 走浏览器 CDP，
   优先找内部 JSON 接口一次取全量。本机 CDP 连接姿势见项目记忆或 tools/cdp.mjs 用法。
4. 采完把该馆 verified 改成 true，并在采集指南.md 的省份进度表打勾。

不要动别的省的文件，也不要跑 normalize.py / build_site.py（归一化和建站由主控统一做）。
完成后回报：每个省采了几个资源、能否线上办证、有没有采不到的坑。
```

---

## 分派后主控（你或我）要做的收尾
各省文件都齐了之后，在项目根目录跑：
```bash
python3 normalize.py     # 跨馆归一化、生成 data/resources.json
python3 build_site.py    # 生成 site/index.html
```
然后抽查几条核对原文，确认无误再对外分享。
