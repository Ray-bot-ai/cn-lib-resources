# 公立图书馆免费电子资源清单

汇总中国大陆各省级公共图书馆对读者**免费开放的电子/数字资源**，以及**如何注册该馆账号**（重点：能否线上注册，还是只能线下）。

## 它长什么样
一个可以双击打开的网页（`site/index.html`），有两种查看方式，顶部一键切换：

1. **按资源** —— 例如「中国历史文献总库·近代报纸库 → 湖北省图书馆 / XX 图书馆…」，点开某个馆显示：该馆注册教程 + 数字资源栏目链接 + 这个馆的订阅范围。
2. **按图书馆** —— 例如「湖北省图书馆（可线上注册）→ 它开放的资源清单」。

## 数据放在哪（三层结构）
| 路径 | 内容 |
|---|---|
| `data/libraries/<省>.json` | **一个图书馆一个文件**：注册方式 + 该馆拥有的资源清单 |
| `data/resources.json` | **资源总目录**：每个资源的规范名、厂商、分类，唯一 ID |
| `build_site.py` | 读上面两者，生成 `site/index.html` |

> 为什么一个馆一个文件：多个智能体并行采集时各写各的，不会互相覆盖。

## 关键规则：什么算「同一个资源」
- 不同镜像网址、但其实是同一个产品 → **合并**成同一个 `canonical_id`。
- 订阅范围不同（比如一个馆买了全库、一个馆只有部分）→ **拆成不同**的资源条目。
- 这个判断在「归一化」这一步统一做（见 `采集指南.md`），采集时先照抄原名即可。

## 怎么生成/更新网页
```bash
# 首次：装 uv（若没有）后建环境
uv venv
# 每次改完数据，重新生成网页：
uv run build_site.py
# 然后双击打开 site/index.html
```
（`build_site.py` 只用标准库，不需要装第三方包。）

## 采集进度
见 `采集指南.md` 里的省份清单勾选表。**所有 `"verified": false` 的字段都还没核实**，不要当成事实。

## 数据来源与免责
资源可用性、订阅范围、注册政策会随时间变化，以各馆官网为准。本清单仅作导航。
