# 图书馆网站结构侦察教程

> 给会开浏览器、但不擅长"看懂网站结构"的采集智能体。
> 目标：拿到一个图书馆的**数字资源栏目**后，搞清楚①资源列表藏在哪 ②怎么一次拿到全部 ③怎么确认没漏。
> 全程用仓库里的 `tools/cdp.mjs` 和 `tools/capture.mjs`（先确保专用 Chrome 已按记忆 `cdp-chrome-remote-debugging-setup` 启动）。

---

## 心法：先侦察，后抓取。不要一上来就抓，先花 1 分钟判断网站属于哪种。

抓之前必须回答三个问题：
1. **列表在哪**：资源名是服务器直接吐在 HTML 里（SSR），还是 JS 异步加载（SPA）？
2. **怎么拿全**：靠翻页？靠滚动懒加载？还是有个接口一次返回全部？
3. **怎么算全**：官网有没有"共 N 个"？翻到什么时候算到头？

---

## ⛔ 头号铁律：判断 SSR 只认 `curl`，绝不用浏览器读到的结果下结论

`curl` 拿的是**服务器原样吐出的 HTML**，不跑任何 JS，结果**确定、可复现**。
浏览器里读到的（`document.documentElement.outerHTML`、CDP `Network.getResponseBody`）会被 JS 影响：

> **真实案例（首都图书馆，workbuddy 栽在这）**：服务器每次都吐完整 SSR HTML（20条/282KB，`curl` 稳定拿到）；但页面 Nuxt 脚本 hydration 后会把 SSR 内容**冲掉**，渲染后的 DOM 变成 10KB 空壳、0 条，`getResponseBody` 也会因响应体被驱逐而**偶发读到 0**。workbuddy 用浏览器读到 0，就误判成"纯 SPA 无 SSR"，于是放弃了本可 `curl` 直接全量的路子。

**规则：**
- 想知道"有没有 SSR / 服务器给了什么" → **只跑 `curl`**。curl 有内容 = 有 SSR = 直接 curl 抓，别碰浏览器。
- **浏览器读到空壳 ≠ 没有 SSR**，只代表"渲染后的前端状态是空的"。这两件事不是一回事。
- 浏览器（CDP）只在 **curl 确实拿不到数据**（真 SPA、要登录态、要交互）时才用。

---

## ⚠️ 先排除"境外 IP 被封"——别把 IP 封锁误当成"网站没有清单"

有些省级馆（如**山西 `lib.sx.cn`**）对**境外 IP 做地理封锁**：首页 HTML 能加载，但所有 `/api/` 数据接口返回 **403 Forbidden（页面里会写 `Client IP: x.x.x.x`）**，导致 SPA 拿不到数据、渲染成空壳，或"时好时坏"。这**极像**"纯 SPA 无接口/无清单"，但根因完全不同。

**症状 → 先查出口 IP：**
```bash
# 在采集用的浏览器里执行
node tools/cdp.mjs eval "$TID" "fetch('https://ipapi.co/json/').then(r=>r.json()).then(j=>j.ip+' '+j.country_name)"
```
- 若 IP 不在中国（本机曾见日本东京 VPN 出口）→ **关掉 VPN/代理、用中国大陆直连**再采。多数省馆的 API 一旦 IP 放行就能直接返回全量清单。
- 判据：接口 403 + 首页 HTML 能开 + 出口 IP 境外 = 几乎肯定是地理封锁，不是结构问题。**这种情况不要下"非全量/无清单"的结论**，标注"待国内 IP 重采"。

## Step 0：先 curl 看原始 HTML，判断 SSR 还是 SPA（决定性的一步）

```bash
# 把资源页 URL 换进去。-L 跟随跳转，-A 伪装浏览器
curl -sL "https://某馆/资源页" -A "Mozilla/5.0" > /tmp/probe.html
wc -c /tmp/probe.html                      # 看有多大
grep -oE '(资源|数据库|item|card|list)' /tmp/probe.html | sort | uniq -c | sort -rn | head
# 关键：随便找一个你在页面上肉眼看到的资源名，grep 它
grep -c "中国知网" /tmp/probe.html         # 换成该页真实出现的某个资源名
```

**判断：**
- 原始 HTML 里 **grep 得到真实资源名** → **SSR 型**，走 **A 路线**（最省事，`curl` 就能全量，不用浏览器）。
- 只有空壳、脚本标签、grep 不到任何资源名 → **SPA 型**，走 **B 路线**（要浏览器）。

> ⚠️ 别凭"官网很现代/是 Vue/Nuxt"就断定它是 SPA。首都图书馆看着像纯 SPA，实际资源页是 SSR，curl 直接能抓。**永远先 curl 验一下**。

## A 路线：SSR 型（如 首都图书馆 clcn.net.cn/resources）

### A1. 找分页参数
资源页 HTML 里搜分页线索：
```bash
grep -oE '[?&]page=[0-9]+' /tmp/probe.html | sort -u        # 常见是 ?page=N
grep -oE '/detail\?id=[0-9]+' /tmp/probe.html | head        # 详情页 id，去重要用它
```
数一页有几条：`grep -c 'item item-cat' /tmp/probe.html`（把 `item item-cat` 换成该站条目的重复标记）。

### A2. 逐页抓，**并行**更快（服务器可能很慢）
```bash
seq 1 30 | xargs -P 8 -I{} sh -c 'curl -s --max-time 30 "https://某馆/资源页?page={}" -A "Mozilla/5.0" > /tmp/p{}.html'
```

### A3. ⚠️⚠️ 最关键的坑：**超出真实页数后，服务器可能重复返回最后一页**
首图第 7 页起一直重复第 6 页（每页都是满满 20 条），不去重就会以为"翻不完"或抓不全。
**解法：按详情页 id 去重，某页"新增去重=0"就说明到头了。**
```bash
python3 - <<'EOF'
import re,glob,json
seen={}; 
for f in sorted(glob.glob('/tmp/p*.html'), key=lambda x:int(re.search(r'p(\d+)',x).group(1))):
    p=int(re.search(r'p(\d+)',f).group(1))
    h=open(f,encoding='utf-8',errors='ignore').read()
    blocks=h.split('item item-cat')[1:]         # 换成该站条目分隔标记
    new=0
    for b in blocks:
        m=re.search(r'detail\?id=(\d+)"[^>]*title="([^"]*)"',b)   # 换成该站的名称/id 模式
        if not m: continue
        rid,name=m.group(1),m.group(2).strip()
        btns=[t for t in ['馆内访问','馆外访问','试用'] if t in b[:2500]]
        if rid not in seen:
            seen[rid]={"id":rid,"name":name,"btns":btns}; new+=1
    print(f"page {p}: 条目{len(blocks)} 新增去重{new}")
    if blocks and new==0: break        # 到头了
print("真实全量 =", len(seen))
json.dump(list(seen.values()), open('/tmp/result.json','w'), ensure_ascii=False, indent=1)
EOF
```

## B 路线：SPA 型（如 上海 z.library.sh.cn、浙江 zjlib.cn）

### B1. 用浏览器打开，确认能渲染
```bash
cd tools
TID=$(node cdp.mjs new "https://某馆/资源页"); echo $TID
sleep 6
node cdp.mjs eval "$TID" "document.querySelectorAll('tr,li,.item,.card').length + ' 个候选条目'"
```

### B2. 优先找内部 JSON 接口（比抓 DOM 省事得多）
资源列表几乎都是 JS 从某个接口取的。两种找法：
```bash
# 法一：看页面加载过的请求
node cdp.mjs eval "$TID" '(function(){return performance.getEntriesByType("resource").map(e=>e.name).filter(n=>/api|list|page|database|resource|db|query|content/i.test(n)&&!/\.(png|jpg|css|js|svg|woff|gif)/i.test(n)).join("\n")})()'

# 法二：抓包（更全，能抓到 performance 漏掉的 XHR/POST）
node capture.mjs "$TID" "api" "https://某馆/资源页" 2>&1 | grep -E "^=== (GET|POST)|REQ_BODY|RESP_LEN"
```
在输出里找形如 `.../pageList`、`.../navigation/all`、`.../digitalResource` 的接口——那就是数据源。
**排除干扰**：`sdk.` / `tongji.` / `matomo` / `hm.baidu` / `predefine_pageview` 这些是**埋点统计**，不是数据，别被骗。

### B3. 直接调接口，pageSize 调大一次取全
```bash
# GET 接口（如浙江）：把 size 调大
node cdp.mjs eval "$TID" 'fetch("https://某馆/……/pageList?current=1&size=500&…").then(r=>r.json()).then(j=>{var L=j.data.records||j.data.list||[];return "total="+(j.data.total)+" 拿到="+L.length+" 字段="+Object.keys(L[0]).join(",")})'
```
拿到后逐条取：`名称`(collectionName/title/name)、`访问URL`(sysPlatformUrl/url)、`访问方式`(有就取)。

### B4. 接口调不动就退回读 DOM（如上海：接口带每次变化的签名头 X-API-KAY，手工调会被拒）
滚动到底把全部条目加载出来，再从渲染后的 DOM 提取：
```bash
node cdp.mjs eval "$TID" '(async function(){
  const q=()=>document.querySelectorAll("tbody tr");   // 换成该站条目选择器
  let last=0,stable=0;
  for(let i=0;i<50&&stable<4;i++){window.scrollTo(0,document.body.scrollHeight);await new Promise(r=>setTimeout(r,700));let n=q().length;if(n===last)stable++;else{stable=0;last=n;}}
  return [...q()].map(tr=>{
    const a=tr.querySelector(".title a,a");                       // 名称+链接
    const btns=[...tr.querySelectorAll("button,a,span")].map(b=>b.innerText.trim()).filter(t=>/馆内访问|馆外访问|链接直达|OA/.test(t));
    return (a?a.innerText.trim():"")+"\t"+[...new Set(btns)].join(",");
  }).join("\n");
})()'
```

## 通用技巧

### 怎么在一大坨 JSON 里找到"资源数组"
```bash
node cdp.mjs eval "$TID" 'fetch("接口URL").then(r=>r.json()).then(j=>{
  function find(o,p){if(Array.isArray(o)&&o[0]&&(o[0].name||o[0].title||o[0].collectionName))return p+" 字段:"+Object.keys(o[0]).join(",")+" 长度:"+o.length;
  if(o&&typeof o=="object")for(var k in o){var r=find(o[k],p+"."+k);if(r)return r}return null}
  return find(j,"")})'
```

### 怎么找"馆内/馆外"访问方式
- 页面上常是每个资源旁的按钮："馆内访问 / 馆外访问 / 链接直达 / OA / 试用"。
- **从馆外的浏览器看到的按钮，就代表馆外能不能访问**：有"馆外访问"=远程可用；只有"馆内访问"=仅到馆。
- 接口/页面里**没有**这个信息就**留空**，绝不猜、绝不从 URL 形状推断（见采集指南铁律）。

### 怎么找办证政策（能否线上办证）
1. 在「服务 / 指南 / 读者指南」里找"办证须知 / 注册指南 / 办证指南"。
2. 正文可能是图片或动态加载，读不到文字时，看有没有**自助注册表单/入口**（凭身份证在小程序/支付宝/官网能自助办证 ≈ 支持线上）。
3. 已见模式：微信小程序办电子证(首图)、支付宝信用注册(浙江)、粤读通小程序(广东)、官网+小程序(上海)。

## 验收清单（防"抓到了但不全"——这是本项目最忌讳的）

抓完每个馆，逐条自问：
- [ ] 官网有没有"共 N 个 / 数据库总量 N"的声明？我抓到的条数**对得上**吗？对不上要查为什么。
- [ ] （SSR分页）我是不是翻到"新增去重=0"才停的？还是翻到某页就自己停了？
- [ ] （接口）total 字段和我实际拿到的条数一致吗？pageSize 是不是够大、没被截断？
- [ ] 随机点开 2 个 `access_url`，确认真能通向该资源（哪怕要登录）。
- [ ] `registration.online` 是**核实**的还是猜的？——这是核心字段，必须核实。

**只要有一条打不了勾，就还没抓完，别急着 verified=true。**
