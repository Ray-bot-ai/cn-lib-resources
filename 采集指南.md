# 采集指南（给负责采集的智能体 / 也给自己）

你的任务：为分配到的省级公共图书馆，采集**免费电子资源**与**注册方式**，写成一个 JSON 文件放到 `data/libraries/`。

## 采集口径（已确认）
- **资源要全量**：把该馆「数字资源/数据库」栏目下能列出的资源尽量都收，不是只挑几个代表。
- **馆内资源也收**：只能到馆用的资源一并记录。
- **`access_method` 有就抓、没就留空**：官网/门户明确标了访问方式（如"馆内访问/馆外访问/OA"按钮）就照实填；没标就**留空**，不要逐个去试、更不要猜。
- **最重要的字段是"能否线上办证"**（`registration.online`）——这是本项目核心，务必核实。

## 铁律
1. **只填你亲眼在官网上核实到的信息。** 没查到就留空 `""` 或 `unknown`，**绝不编造**——尤其是"能否线上注册"。
   - **`access_method`（馆内/馆外）只填官网明确标注或实测的结果，不许从 URL 形状推断。** 镜像站 ≠ 只能到馆——很多镜像支持远程访问；内网 IP(10.x) 只作客观备注，不能据此断言"仅馆内"。列表接口若没有访问权限字段，就留空。（教训：首采曾把"内网镜像"错标成"到馆访问"。）
2. **一个图书馆一个文件**，文件名用省份拼音，如 `data/libraries/hunan.json`。不要动别人的文件。
3. 资源名先**照抄官网原样**填进 `raw_name`；`canonical_id` 可以留空，归一化时统一处理。
4. 全部核实完，把该馆 `"verified"` 改成 `true`。

## 文件格式（照着 `data/libraries/_template.json` 抄）
```json
{
  "id": "hunan-prov",
  "name": "湖南图书馆",
  "level": "省级公共图书馆",
  "region": "湖南省",
  "verified": true,

  "official_site": "官网首页 URL",
  "digital_resource_url": "「数字资源/电子资源」栏目页 URL",

  "registration": {
    "online": "yes | no | partial | unknown",
    "eligibility": "谁能注册，如『全国读者』『需本省身份证』『需到馆持证』",
    "cost": "免费 / 免费但需押金 / …",
    "how": "一句话说清怎么注册（线上入口在哪 / 线下带什么去哪）",
    "tutorial_url": "官方注册教程/办证指南页 URL（没有就留空）",
    "notes": ""
  },

  "resources": [
    {
      "canonical_id": "",
      "raw_name": "官网上写的资源名，原样照抄",
      "access_url": "该资源的访问入口 URL",
      "scope": "订阅范围，如『全库』『仅XXXX-XXXX年』『试用』",
      "access_method": "馆内 / 馆外凭读者证 / 公网免登录",
      "notes": ""
    }
  ]
}
```

## `online` 四个取值的判断
- `yes`：能全程在网上（官网/App/公众号/支付宝等）注册并立即用电子资源。
- `partial`：能线上注册但功能受限（如线上只能用部分库，完整权限需到馆）。
- `no`：必须本人到馆办证才能用。
- `unknown`：官网没写清、你没核实到。**存疑一律填 unknown，别猜。**

## 采集步骤建议
1. 打开省级图书馆官网，找「数字资源 / 电子资源 / 数据库」栏目 → 记 `digital_resource_url`。
2. 逐个记下资源：原名、访问链接、订阅范围、是否需要馆内/读者证。
3. 找「办证指南 / 读者指南 / 注册」→ 判断 `online` 并记 `tutorial_url`。
4. 抽查：随便点开 2 个资源链接，确认确实能通向该资源（哪怕要登录）。

## 技术提示（采集时会遇到的坑）
- **省级馆官网基本都是 SPA**（Vue/Nuxt/Angular）：资源列表靠 JS 动态加载，`curl`/静态抓取只能拿到空壳。必须用**真实浏览器**（加载 web-access skill，走浏览器 CDP）。
- **优先找内部 JSON 接口**：在资源页打开后看 `performance.getEntriesByType("resource")` 或浏览器 Network，多能找到形如 `.../digitalResource/pageList` 的接口，一次返回全量，比翻页省事。已知：浙江 `zjlib.cn/bff-api/.../digitalResource/pageList`（GET）；上海 `z.library.sh.cn/api/dbs/navigation/all`（POST，且带每次变化的签名头 `X-API-KAY`，直接调会被拒——改从渲染后的页面 DOM 或响应抓取）。
- 本机 CDP 连接的正确姿势见用户记忆 `cdp-chrome-remote-debugging-setup`（默认配置目录连不上，要用独立 `--user-data-dir`）。
- **别急着判"纯SPA无接口"**：有些馆（如首都图书馆 `clcn.net.cn/resources`）资源页其实是 **SSR**，`curl` 直接能拿到条目——先 `curl` 看原始 HTML 里有没有 `item`/资源名，再决定要不要上浏览器。
- **SSR 分页站的坑**：用 `?page=N` 逐页 `curl`。注意**超出真实页数后服务器可能重复返回最后一页**（首图第 7 页起就重复第 6 页），会让人误以为无穷多页或抓不全——**务必按详情页 id 去重**，当某页"新增去重=0"就说明到头了。首图真实全量=120（6页×20）。
- **办证政策**常在「服务/指南 → 办证须知/注册指南」，正文可能是图片或动态加载；找不到文字就看有没有**自助注册表单**（有实名注册入口≈支持线上办证）。

## 归一化（这一步统一做，采集者不用管）
把各馆 `resources[].raw_name` 对照，指向同一产品的：
- **镜像相同** → 用同一个 `canonical_id`，并在 `data/resources.json` 建一条。
- **订阅范围不同** → 拆成不同 `canonical_id`（如 `xxx-full` / `xxx-1949`）。
然后把每条 holding 的 `canonical_id` 回填到各馆文件里。
